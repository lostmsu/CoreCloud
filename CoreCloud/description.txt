Задача

	Задачей данной работы является разработка комплекса программных средств для создания инфраструктуры децентрализованных распределённых вычислений с добровольным участием. Комплекс должен быть разбит на клиентскую (клиент ДУ) и сервисную (сервис ДУ) части, каждая из которых в свою очередь должна быть разбита на несколько уровней.


Терминология

Добровольный участник - компьютер, ресурсы которого добровольно предоставлены для распределённых вычислений.
Сервис ДУ - программное средство, обеспечивающее публикацию и доступ к ресурсам, предоставленным для распределённых вычислений.
Клиент ДУ - программное средство, обеспечивающее поиск ресурсов и задействование их при решении задач.

Ожидаемый результат - реализация большей части тебований.

Каждый пункт требований не помеченный вопросами уже имеет минимальную реализацию. Множество всех таких пунктов - минимальные требования.
Пункты, помеченные вопросом можно реализовать в дополнение к минимуму, но каждый из них не является необходимым. По ним есть конкретные идеи, как это сделать (кроме 2.2, его нужно будет обсудить). Для них надо расставить приоритеты.
Если какой-то пункт помечен вопросом, следует считать, что все его подпункты также помечены вопросом.

К началу экзаменационной сессии могу реализовать минимальные требования и, возможно, 1 из дополений. Параллельно существующей задаче модификации агента, как мы и договаривались.
